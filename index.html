关于客体恒常性的总结

前段时间看了一些关于客体恒常性的书籍《依恋》《人格心理学》，想结合看完的浅谈下写下一些备忘总结

简单介绍一下客体恒常性的概念，大概意思就是，我们和内心的”客体”有一种安宁恒定的状态，就好比对于父母离开的家的独处的孩子来说。

如果正常稳定的客体，会让我们觉得父母虽然不在，孩子虽然会想父母，但是他会感到他和父母心里仍然在一起。

反之缺失的话会产生会焦虑、焦躁、害怕被抛弃的情绪。

造成客体缺失的原因有很多，童年不友好的客体分离体验，成长经历中被忽冷忽热的对待，亲朋好友的逝世。

目前了解和学习客体恒常性就我自身而言独处时感到安宁有很大的帮助，我不再像之前那样独处时会有很多侵入性的想法。

关于客体恒常性的定义？

客体和主体呈相对视角，主体也就是我们的内心视角感，而客体意指我们主体的内心视角所看到依恋目标所指的对象。

而恒常性就比较字面意思了，是我们理解了依恋目标，且知道对我们充满爱意，与我们较好的维系情感上的链接时，我们会把

依恋目标转换为一个”内在的客体形象”，即便监护人，伴侣暂时离开了我们，我们也知道他们只是暂时的离开了我们的视野

没有抛弃我们或者消失了，我们仍然知道我们是被爱的，被支持的。

Hexo搭建部署至Github

最近折腾Blog折腾到了Hexo上，写这篇文章主要目的分享自己部署在Github上的坑，另外一个原因是Hexo是本地推送然后生成静态页面，搞不好那天本地环境折腾出问题了还得重新来过，所以记录下部署过程以防备忘（健忘）

环境配置

安装Git

安装Git，下载完成后基本一路点到底，安装完成后右键菜单会有Git Gui Here 和 Git Bash Here

安装Node.js

因为Hexo是基于Node.Js的环境，所以我们需要安装Node.Js以方便本地调试，官网的Node.Js因线路问题安装包下载较慢

建议使用NodeJs中文网安装最新的”长期支持版本”即可，点击后根据本机系统选择相应的安装包下载完成后安装即可

安装完成后在Cmd 输入 node-v 提示安装版本号（V16.16.0）即代表安装成功
