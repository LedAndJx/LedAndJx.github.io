Hexo搭建部署至Github

最近折腾Blog折腾到了Hexo上，写这篇文章主要目的分享自己部署在Github上的坑，另外一个原因是Hexo是本地推送然后生成静态页面，搞不好那天本地环境折腾出问题了还得重新来过，所以记录下部署过程以防备忘（健忘）

环境配置

安装Git

安装Git，下载完成后基本一路点到底，安装完成后右键菜单会有Git Gui Here 和 Git Bash Here

安装Node.js

因为Hexo是基于Node.Js的环境，所以我们需要安装Node.Js以方便本地调试，官网的Node.Js因线路问题安装包下载较慢

建议使用NodeJs中文网安装最新的”长期支持版本”即可，点击后根据本机系统选择相应的安装包下载完成后安装即可

安装完成后在Cmd 输入 node-v 提示安装版本号（V16.16.0）即代表安装成功

Git仓库和SSH配置

关联Github账号&邮箱

在桌面右键，点击Git Bash Here，出现Git命令行弹窗

在弹窗内分别输入以下两行命令，并回车运行

输入完成后一般没有反应然后去本地文件，路径为 C:/Users/[username]/.gitconfig （未找到的话建议开启隐藏文件或换一个用户名尝试搜索文件.gitconfig，找到该文件后打开里面有刚才输入的用户名和邮箱即配置成功

配置Github仓库关联SSH

首先在Github创建一个”用户名.github.io”的仓库，例如：SuperLed.github.io，创建完成后我们关联SSH用来授权本地和仓库的权限

打开Git Bash Here命令行窗口 输入命令：Cd ~/.ssh 来创建一个本地密匙

创建完成后，会在本地生成id_rsa_id_rsa.pub，该文件存放在 C:\Users\admin.ssh
